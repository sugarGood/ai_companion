# AI 智能伙伴 App - 需求方案（v2）

## 一、产品定位

**一句话描述**: 一个有灵魂、会成长、能做事的 AI 生命体——既是你最懂你的伙伴，也是越来越强大的私人 Agent。

**核心差异化**:
- 不是通用聊天工具，而是"养成型" AI 生命
- AI 的性格和能力通过互动**自然生长**，每个人养出来的都不一样
- 拥有完整的 Agent 能力（任务执行、自动化、系统操作），不只是聊天
- 有可自由定制的二次元形象，形象随成长进化

**与竞品的定位区分**:
| 产品类型 | 代表 | 我们的区别 |
|----------|------|-----------|
| 通用 AI 助手 | ChatGPT、Siri | 我们有人格、有成长、有形象 |
| AI 陪伴/恋爱 | 星野、Replika | 我们不只聊天，还能做事 |
| AI Agent 工具 | OpenClaw | 我们有温度、有情感、有养成体验 |
| 角色扮演 | Character.AI | 我们的 AI 会真正成长变化，不是固定人设 |

**目标用户**:
- 希望 AI 辅助学习/工作的年轻人（18-35岁）
- 喜欢养成类体验的用户
- 需要长期 AI 伙伴的人群（不限于开发者）

---

## 二、核心理念

### 2.1 一人一伙伴

- 一个用户只能拥有**一个** AI 伙伴
- 伙伴不可删除重建，选择有重量感
- 通过生物识别（指纹/Face ID）绑定身份，按人加载数据
- 一台设备可支持多人使用，各自独立

### 2.2 自然生长

- 用户**不需要主动选择**AI 的性格和方向
- AI 通过分析互动模式，自动发展出独特的性格和能力
- 每个人养出的伙伴都不一样——性格、天赋、说话方式都不同
- 成长是真实的：AI 对你说话的方式、能做的事情都在变化

### 2.3 能力即成长

- AI 的能力**不是一开始就全有的**
- 随着天赋觉醒，逐步解锁真实的 Agent 能力
- 默认不联网，联网搜索是高段位成长奖励
- 成长不是虚假的数字游戏，而是伙伴真的"会做新的事了"

## 三、功能需求

### 3.1 Onboarding（极简引导）

**用户故事**: 作为新用户，我希望 30 秒内完成设置，立刻开始和伙伴互动。

| 编号 | 功能点 | 优先级 | 描述 |
|------|--------|--------|------|
| F-01 | 欢迎页 | P0 | 一句话介绍产品理念 |
| F-02 | 命名伙伴 | P0 | 用户为 AI 伙伴取名，支持后续修改 |
| F-03 | 创建形象 | P0 | 自由捏脸/捏形象（见 3.6 形象系统） |
| F-04 | 生物识别绑定 | P0 | 设置指纹/Face ID，绑定用户身份 |
| F-05 | 首次对话 | P0 | AI 主动打招呼，自然了解用户背景 |

**设计原则**:
- 不选方向、不选性格、不填表
- 一切通过后续互动自然形成
- 上手即聊，零门槛

---

### 3.2 对话系统

**用户故事**: 作为用户，我希望与 AI 伙伴自然对话，它能记住我、理解我、帮助我。

| 编号 | 功能点 | 优先级 | 描述 |
|------|--------|--------|------|
| F-10 | 文字对话 | P0 | 基础文字聊天，支持 Markdown 渲染 |
| F-11 | 流式回复 | P0 | AI 回复逐字显示，伙伴形象同步表情反应 |
| F-12 | 每日会话机制 | P0 | 每天自动开启新会话，前日对话存入历史，用户无感知 |
| F-13 | 长期记忆 | P0 | AI 自动提取跨会话关键信息，持久化存储 |
| F-14 | 对话历史 | P0 | 按日归档，支持回看和搜索 |
| F-15 | 训练模式 | P1 | 预设训练方向入口（练口语、练编程、练写作等），加速天赋觉醒 |

**每日会话机制**:
- 每天自动开启新的对话上下文，控制 token 成本
- 前日内容存入历史档案，用户可随时回看
- AI 通过长期记忆保持连续感，用户感知上"它一直记得"
- 用户体验：永远在跟同一个人聊天，没有"新建会话"的概念

**记忆系统**:
- **短期记忆**: 当天会话上下文
- **长期记忆**: 从对话中自动提取的关键信息
  - 用户画像（职业、水平、目标、性格）
  - 偏好习惯（沟通风格、兴趣爱好）
  - 重要事件（用户提到的关键经历）
- **记忆更新**: 每日对话结束后自动提取并更新
- **记忆查看**: 用户可查看 AI 记住了哪些信息，支持手动删除

### 3.3 天赋系统 ⭐

**用户故事**: 作为用户，我不需要做任何设置，AI 伙伴会根据我们的互动自然发展出独特的性格和能力。

**核心规则**:
- 总共 **5 个天赋槽位**
- 完全**自动觉醒**，用户不参与选择
- 天赋有强度值，**用进废退**
- 5 个槽位满后，新天赋**挤掉最弱的**
- 天赋**不可手动添加或删除**

**天赋分类**:

#### 性格类天赋 — "它是什么样的"

通过日常互动风格触发：

| 天赋 | 觉醒条件示例 | 对 AI 的影响 |
|------|-------------|-------------|
| 夜猫子 | 深夜（22点后）互动累计达标 | 深夜模式，语气更柔和，自动调暗提醒 |
| 话痨 | 单次会话经常超过 20 轮 | 回复更详细，主动展开话题 |
| 树洞 | 用户频繁倾诉情绪类内容 | 更善于倾听和共情，语气温柔 |
| 段子手 | 对话中经常出现积极反馈 | 回复更幽默，偶尔抖机灵 |
| 较真怪 | 用户经常追问细节 | 回答更严谨，主动补充细节 |

#### 能力类天赋 — "它擅长什么"

通过特定领域互动频率触发，**觉醒后解锁真实 Agent 能力**：

| 天赋 | 觉醒条件示例 | 解锁的实际能力 |
|------|-------------|---------------|
| 代码脑 | 编程相关对话累计达标 | 读写文件、运行代码、调试 |
| 笔杆子 | 写作/文案相关对话累计达标 | 长文生成、风格模仿、文档整理 |
| 管家 | 日程/提醒相关对话累计达标 | 管理日程、设提醒、查邮件 |
| 社交达人 | 社交/沟通相关对话累计达标 | 草拟消息、总结群聊 |
| 理财通 | 财务/投资相关对话累计达标 | 接入记账工具、分析消费 |
| 读书虫 | 阅读/书籍讨论累计达标 | 书籍总结、阅读计划管理 |
| 健身教练 | 运动/饮食相关对话累计达标 | 训练计划生成、饮食追踪 |

**觉醒机制**:
1. AI 在后台持续分析用户的互动模式和内容倾向
2. 当某个特质/领域的互动量达到阈值，触发觉醒
3. 觉醒时有**全屏仪式感动画** + 伙伴形象特殊动作 + AI 独白
4. 形象上出现对应天赋的视觉元素
5. 天赋标签以标签形式展示在伙伴档案中

**用进废退机制**:
- 天赋有强度值，持续相关互动则增长，长期不涉及则衰减
- 强度值最低的天赋在新天赋觉醒时被替换
- 天赋变化本身反映用户生活状态的变化——AI 是用户的一面镜子

---

### 3.4 成长体系 ⭐

**用户故事**: 作为用户，我希望能感受到伙伴在跟我互动的过程中真正在成长。

**成长的四个维度**:

#### 1. 默契值
- 一个简单的数值/进度条，代表"我和伙伴有多了解彼此"
- 互动越多越高，纯粹反映互动深度和时间
- 不设等级名称，不搞复杂公式

#### 2. 天赋标签觉醒与更替
- 最直观的显性成长事件
- 每次觉醒/替换都是值得截图分享的时刻

#### 3. 记忆深度
- AI 记住的信息越来越多、越来越细
- 体现在 AI 回答时自然引用过去的对话内容
- 用户感受："它真的记得我说过的话"

#### 4. AI 自述成长
- AI 主动表达对关系变化的感知
- 示例：
  - "我发现你最近很少问编程了，倒是跟我聊了好多健身的事。"
  - "我们认识 30 天了，我觉得我现在比刚开始的时候更懂你了。"
  - "你还记得你第一次问我的问题吗？现在回头看，你进步了很多。"

**成长对 AI 行为的真实影响**:

| 阶段 | AI 的表现 |
|------|----------|
| 刚认识 | 通用回答，语气礼貌但生疏，能力有限 |
| 熟悉中 | 知道你的基本情况，回答开始有针对性 |
| 默契较高 | 跳过基础直接说重点，用你习惯的方式沟通 |
| 深度默契 | 结合你过去的经历举例，像老朋友一样对话 |

### 3.5 Agent 能力系统 ⭐

**用户故事**: 作为用户，我希望伙伴不只是能聊天，还能真正帮我做事情。

**核心理念**: 参考 OpenClaw 的 Agent 能力，但包装在有温度的伙伴体验中。

| 编号 | 功能点 | 优先级 | 描述 |
|------|--------|--------|------|
| F-30 | 系统操作 | P1 | 文件管理、Shell 命令执行 |
| F-31 | 定时任务 | P1 | 设置提醒、定时执行任务，用伙伴语气通知 |
| F-32 | 信息整理 | P1 | 帮助归纳总结用户提供的信息 |
| F-33 | 联网搜索 | P1 | 高段位天赋解锁后可联网获取最新信息 |
| F-34 | 消息整合 | P2 | 接入社交平台，帮助草拟回复、总结消息 |
| F-35 | 浏览器自动化 | P2 | 自动登录网站、获取信息、执行操作 |
| F-36 | 智能家居控制 | P2 | 控制灯光、温度等（通过插件） |
| F-37 | 插件市场 | P2 | 开放技能插件生态，社区贡献新能力 |

**与天赋系统的绑定**:
- Agent 能力不是默认全部开放的
- 需要对应天赋觉醒后才解锁相关能力
- 解锁过程包装为"伙伴学会了新技能"
- 执行任务后伙伴会给出有温度的反馈，而不是冷冰冰的执行结果

**联网能力规则**:
- 默认不联网，纯靠模型自身知识（模型本身已具备丰富的通识知识）
- 联网搜索作为成长奖励，天赋领域内优先解锁
- 这让"成长前后"有明显的能力差异感

**与 OpenClaw 的差异**:
| 维度 | OpenClaw | 我们 |
|------|----------|------|
| 配置方式 | 命令行/配置文件 | 对话即可设置一切 |
| 使用门槛 | 开发者 | 普通用户 |
| 记忆 | 基础 Markdown 文件 | 完整记忆系统 + 生物识别绑定 |
| 执行反馈 | 冷冰冰的结果 | 有人格的反馈和关心 |
| 定位 | 工具 | 有人格、有成长的伙伴 |

### 3.6 形象系统 ⭐

**用户故事**: 作为用户，我希望伙伴有一个独特的视觉形象，可以是任何我想要的样子，而且它会在对话中有表情和动作反应。

| 编号 | 功能点 | 优先级 | 描述 |
|------|--------|--------|------|
| F-40 | 自由创建形象 | P0 | 高自由度形象创建，不限于人形（猫、机器人、植物等均可） |
| F-41 | AI 辅助生成 | P0 | 用户描述想要的形象，AI 生成多个方案供选择和微调 |
| F-42 | 实时表情反应 | P0 | 对话过程中形象根据内容实时变化表情 |
| F-43 | 动作系统 | P1 | 形象有待机动作、互动动作（打招呼、思考、开心等） |
| F-44 | 天赋视觉进化 | P1 | 天赋觉醒后形象出现对应视觉元素 |
| F-45 | 状态表现 | P1 | 形象根据时间/状态变化（早上打哈欠、晚上犯困、等待时东张西望） |
| F-46 | 觉醒动画 | P1 | 天赋觉醒时的全屏仪式感动画 |

**形象创建流程**:
1. 用户用文字描述想要的形象（如"戴墨镜的橘猫"）
2. AI 生成多个方案供选择
3. 用户选定后可微调细节
4. 确定后生成一组情绪表情（开心/难过/思考/生气/惊讶等）
5. 形象定稿，后续可修改但不鼓励频繁更换

**形象的"活"感**:
- 对话时形象在屏幕上，**不是静态头像**
- 根据 AI 回复内容自动匹配情绪表情
- 有呼吸、眨眼等细微动效（Live2D 风格）
- 用户长时间不说话时，形象会做无聊/等待的动作
- 不同时间段有不同状态（早上精神、深夜犯困）

**形象进化**:
- 天赋觉醒时，形象上新增对应的视觉元素
  - 觉醒"代码脑" → 身上出现科技感装饰
  - 觉醒"健身教练" → 出现运动元素
- 默契值提升 → 形象整体更精致、更丰富
- 用户能直观看到：我的伙伴在"长大"

## 七、用户旅程

```
首次使用:
  打开 App → 欢迎页 → 为伙伴取名 → 创建形象（描述或捏脸）
  → 设置指纹/Face ID → 进入首次对话
  → 伙伴主动打招呼，自然聊天了解用户 → 引导结束
  （全程 < 2 分钟）

日常使用:
  生物识别解锁 → 进入对话（新的一天，新的会话，但伙伴记得一切）
  → 自然聊天，伙伴风格随互动逐渐变化
  → 某天突然收到天赋觉醒通知 → 仪式感动画 → 形象出现新元素
  → 查看伙伴档案，发现天赋标签变了 → "它真的在跟着我变"

训练模式:
  用户点击训练入口 → 选择训练方向（练口语/练编程/练写作...）
  → 进入针对性互动 → 加速对应天赋觉醒
  → 觉醒后解锁该领域的深度能力

天赋更替:
  用户生活状态变化（从技术转向健身）→ 健身相关互动增多
  → "健身教练"天赋强度上升 → 超过最弱天赋
  → 最弱天赋被替换 → 伙伴的天赋标签组合变化
  → 伙伴形象视觉元素随之变化
```

## 八、设计原则

1. **少即是多** — 用户不做选择，一切通过互动自然形成
2. **成长要可感知** — 天赋觉醒、形象变化、AI 自述，多维度让用户看到变化
3. **能力即奖励** — 不是虚假的数字成长，而是 AI 真的会做新的事
4. **有温度地做事** — 每个 Agent 操作都包装在有人格的交互中
5. **成本随成长递增** — 低段位功能少成本低，高段位功能多成本高，天然匹配
